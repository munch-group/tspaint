Code organization and data flow

This page is the map a reader or contributor needs to navigate the tspaint source: how the modules fit together, what flows between them, and which invariants keep the result correct. It documents the code paths, not the science — the authoritative model and its justification live in CLAUDE.md (referenced below as §N). Names here are verified against src/tspaint/__init__.py; links go to the API reference where one exists.

The one call

The whole library funnels through a single high-level entry point:

import tspaint

ts     = tspaint.simulate_admixture(n_admix=10, n_ref=10)   # or tspaint.io.infer_tree_sequence(...)
labels = {0: 0, 1: 0, 2: 1, 3: 1}                         # reference sample-node -> ancestry state
painting = tspaint.paint(ts, labels)                        # EM-fit on references, paint the queries

painting.posteriors[q]            # soft per-position posterior for query q (list[Segment])
painting.segments(deadband=0.4)   # hard ancestry tracts for downstream dating

paint ties three stages together — fit, paint, segment — and returns a Painting. Everything else in the package is either machinery underneath those stages or tooling around them (front ends, comparators, metrics, experiment drivers).

Walkthrough

0. Get a tree sequence. tspaint paints any tskit.TreeSequence whose nodes keep stable IDs across marginal trees (§5). The tspaint.io namespace gathers the supported ways to obtain one from genotypes; a true/simulated ARG works directly.

1. Emissions (§2.2). build_emissions (in em.py) turns the labels into per-sample Felsenstein emission vectors. A labelled reference tip i gets the soft clamp w_i · 1[label] + (1 − w_i) · π (anchors default to w = 1, a hard clamp); every other sample gets the query emission π (flat / root-frequency). This is rebuilt each EM iteration because w and π move.

2. E-step: pruning, per tree, per root (§3.1). pruning.py runs exact two-pass sum-product message passing on each marginal tree. The up-pass (post-order) accumulates partial likelihoods L_u(s) as a product over all children (polytomy-safe); the down-pass (pre-order) yields the posterior marginal γ_u, the joint parent–child posterior ξ_{(p,c)} (a K×K object per branch), a leave-one-out marginal loo_u (the outside message excluding a tip’s own emission — credibility evidence that cannot self-confirm), and the exact tree log-likelihood. A marginal tree may be a forest, so this runs once per root and aggregates.

3. Edge-blocked, span-weighted sufficient statistics (§3.3 — the correctness core). accumulate.py drives the genome with zip(ts.edge_diffs(), ts.trees()). For each marginal tree it prunes once, then banks each entering edge’s contribution exactly once, on entry, weighted by that edge’s own span (right − left). A clade persisting across many trees is one set of wide-span edges, so summing by span partitions the genome without double-counting (the edge-table invariant: the intervals on which a node is a child are disjoint). Root-state mass is accumulated per interval, not per edge. The accumulators are S_dwell, S_jumps, S_root, per-tip S_cred (agree/disagree for the credibility update), and a span-integrated loglik.

4. Per-branch Van Loan rewards (§3.2). For each entering edge, branch_stats.py computes the endpoint-conditioned expected dwell time per state and expected jump counts per ordered pair via the Van Loan block-triangular matrix exponential (Van Loan, 1978; Hobolth and Jensen, 2011). The (Q, t)-dependent part (branch_kernel) is cached by branch length and applied to each edge’s ξ cheaply (stats_from_kernel), so the expensive expm runs once per distinct branch length per iteration, not once per edge. This module is the designated Phasic seam (§12): swap the expm blocks for Phasic without changing callers.

5. M-step (closed form, §3.4). Back in em.py, the pooled statistics give the generator m_step_Q (q_mn = S_jumps[m,n] / S_dwell[m]), the root frequencies m_step_pi, and the per-tip credibility m_step_w (MAP under a Beta(α, β) prior). fit iterates steps 1–5 to a log-likelihood tolerance or max_iter, returning a FitResult (Q, π, {w_i}, loglik_history). The default estimate_pi=False holds π uniform — the robust choice, since π is a prior on the arbitrary GMRCA state and estimating it from washing deep branches is the §6 degeneracy.

6. Deliverable: the soft painting (§2.4). With the fitted (Q, π, w), paint rebuilds emissions and calls posterior_table (in output.py), which runs the down-pass per marginal tree and records each query’s posterior as contiguous Segments covering [0, L). Spans where a sample is an isolated root are tagged MISSING_INFO (the tree carries no information — the posterior is the prior π, distinct from a 50–50 uncertain call). The result is wrapped in a Painting.

7. Optional horizontal smoothing (§7). With paint(..., smooth=True), bp/horizontal.py runs a single-pass genome-axis forward–backward (EP approximation) over each tip’s per-tree beliefs, with a per-breakpoint near-identity transition (switch penalty epsilon). This adds the propagation of uncertainty across persist-but-reparent breakpoints that blocked EM structurally omits — recommended on inferred (tsinfer / Relate) ARGs, redundant on a true ARG.

8. Hard tracts. Painting.segments(deadband) collapses the soft posteriors into hard (left, right, state) tracts via hard_segments. The deadband suppresses low-confidence argmax flips that fragment long tracts — a precision/recall dial a fixed hard segmenter does not expose. This is the object a downstream tract-length / admixture-pulse dating analysis consumes.

A parallel path: admixture dating (tspaint.dating)

Dating is not part of the paint pipeline — it is a separate estimator that reuses the same engine (pruning + Van Loan rewards + edge-blocked accumulation) with one change: the generator becomes time-inhomogeneous, piecewise-constant on a logarithmic time grid. The flow (admixture dating has the math):

  • grid (dating/grid.py) — log_time_grid / split_branch cut each branch at the cell boundaries it crosses.
  • E-step (dating/estep.py) — branch_cell_stats sandwiches the per-cell Van Loan reward between the composite transition products above/below it on the branch, conditioned on the same endpoint posterior ξ; accumulated per edge, span-weighted, into per-cell occupation D and directional jumps J. Summed over cells this collapses to the homogeneous branch_expected_stats (a tested invariant).
  • M-step (dating/mstep.py) — each direction’s rate is a penalised-Poisson log-spline with GCV smoothing, restricted to the informative occupation window (directional_rate_splines).
  • EM loop (dating/em.py) — fit_rate_through_time alternates the two to a monotone log-likelihood, returning a RateThroughTime (q_AB(t), q_BA(t), .plot()).

Entry points: fit_rate_through_time(ts, labels) standalone, or Painting.rate_through_time() which warm-starts from the painting’s fitted (Q, π, w) (skips the homogeneous fit) and returns a new profile — it does not modify painting.posteriors. Feeding Q(t) back into painting (dating.paint_qt) is available but off the default path: it gives no painting-accuracy gain, only sharper confidence at ~4–5× cost, so the paths stay side by side.

Front ends (tspaint.io)

tspaint is front-end agnostic: it paints any tree sequence with cross-tree node-ID stability (§5). The supported routes to one:

Front end Module What it does
tsinfer io_tsinfer infer_tree_sequence runs tsinfer on genotypes → a point-estimate ARG with native node-ID stability. add_mutations overlays variants on a bare ARG first. The realistic substrate where tree accuracy becomes the binding constraint (§9).
SINGER io_singer singer_tree_sequences shells out to the SINGER binary (haploid VCF → singer -ploidy 1, with a retry loop → tskit), returning posterior ARG samples. These represent genuine ARG uncertainty, so they are the input to the ensemble merge (§7.4). Sample order preserved; times in generations.
Relate --compress external (io_relate) Convert outside Python with Convert --mode ConvertToTreeSequence --compress, then tskit.load. --compress is load-bearing — it unifies a persistent clade into one long-span node ID (§5), the invariant the edge-blocking depends on.

RFMix is not a front end — it is genotype-native and appears as a comparator (below).

Comparators (tspaint.compare)

A painter is any f(ts, labels, queries) -> {sample: [Segment]}. The harness scores every painter against the same simulated truth on equal footing, so tspaint, simple baselines, and external tools compare directly.

Painter Module What it is
tspaint_paint compare The full method (EM fit + down-pass posterior) in painter form.
nearest_reference_paint compare ARG-native baseline: paint each query by its nearest labelled reference (smallest TMRCA) per tree. No CTMC, no EM — the genealogy painter the generative model should beat.
rfmix_paint io_rfmix The field-standard segment incumbent (RFMix v2): writes phased VCFs, shells out to rfmix (isolated compare pixi env), parses the .fb.tsv posteriors back to Segment tracks.
head_to_head compare Driver: simulate one admixture scenario, score a panel of painters across ARG substrates ("true", "tsinfer").
score_painter compare Run one painter and return balanced accuracy, accuracy, and mean confidence.

To add a comparator (MOSAIC/FLARE, ARGMix, AncestralPaths, …), implement the same painter signature — shell out to the tool over a VCF, parse its calls into Segments — and pass it to head_to_head; it is then scored like the rest.

Invariants that keep it correct

These come straight from the tskit data model and the method’s design (§4, §12). A naive traversal violates them and silently corrupts the sufficient statistics or the painting — they are the load-bearing rules, not optional polish.

Invariant Where enforced Why
Accumulate per edge, span-weighted, banked once on entry accumulate.py (edges_in loop, w_edge = e.right − e.left) A persistent clade is one wide-span edge set; summing by span partitions the genome without double-counting and is the channel for genome-scale autocorrelation. Never sum per-(tree × branch).
Prune per root pruning.prune_tree (for r in tree.roots) A marginal tree may be a forest; each root carries its own π. There may be no single root.
Skip root branches accumulate.py / pruning.py (root has no parent edge); branch_kernel returns None for t ≤ 0 Root branch length is 0 by definition in tskit; root state enters via π (S_root) only, never via Q on a (nonexistent) root edge.
Polytomy-safe traversal pruning.py (tree.children(u) / sib pointers; product over all children) Internal nodes with >2 children are real and first-class; never assume binarity.
Isolated span = missing-info ≠ uncertain pruning.py (tags missing_info), output.py (MISSING_INFO status, missing_info_mask) An isolated sample’s posterior falls back to the prior and is tagged distinctly; conflating “no information” with “50–50” is a real interpretive error for introgression mapping.
Keep a hard-clamped anchor set em.fit (raises ValueError if soft_refs covers every labelled tip) Never let the whole panel float — that reintroduces unsupervised degeneracy (label switching, Q/w trade-off, collapse).
Input must have cross-tree node-ID stability front ends (tspaint.io); diagnostics (diagnostics.node_persistence) The edge-blocking and horizontal coupling depend on a persistent clade being one node ID across trees. Delivered natively by msprime/tsinfer/SINGER, or by Relate --compress. Verify with the persistence histogram before trusting results.
Leave-one-out for credibility pruning.py (loo), accumulate.py (S_cred) Judge a tip’s label against what the rest of the tree says, not its own self-confirming posterior.

Module reference

Module Responsibility
api.py High-level paint pipeline and the Painting result. The one call most users need.
model.py Ancestry CTMC: generator Q, root frequencies π, transition matrices, tip/query emissions, the soft-credibility noise model. Generator-agnostic; touches no tskit.
pruning.py Felsenstein two-pass sum-product per marginal tree, per root, polytomy-safe. Produces γ, ξ, leave-one-out marginals, missing-info tags, exact log-likelihood. The E-step kernel.
branch_stats.py Per-branch endpoint-conditioned expected dwell times and jump counts via the Van Loan block exponential. Length-cached. The designated Phasic seam.
accumulate.py The correctness core: edge_diffs sweep, edge-blocked span-weighted sufficient statistics, banked once per edge.
em.py EM orchestration (fit) + the closed-form M-step (m_step_Q/_pi/_w) + build_emissions.
output.py The deliverable: per-haplotype, per-position posterior as Segment tracks; hard_segments collapse; missing_info_mask; posterior_at.
bp/horizontal.py Deferred §7 horizontal coupling: single-pass genome-axis forward–backward (EP) over per-tree beliefs; bp_paint, bp_smooth, bp_smooth_track. Helps on inferred ARGs.
dating/grid.py Log-time grid for the time-inhomogeneous estimator: log_time_grid, cell_centers, split_branch (cut a branch at cell boundaries).
dating/estep.py Per-cell endpoint-conditioned stats (branch_cell_stats, Van Loan sandwiched by composite transitions), time-binned accumulation, and the Q(t) painter paint_qt.
dating/mstep.py Directional penalised-Poisson log-spline rate fit with GCV smoothing: fit_poisson_spline, select_lambda_gcv, directional_rate_splines.
dating/em.py The time-inhomogeneous EM (fit_rate_through_time) → RateThroughTime; make_Q_of_cell. The tspaint.dating headline — a parallel path to paint, not part of it.
ensemble.py Marginalise ARG uncertainty: average per-position paintings across a posterior ensemble (e.g. SINGER) → MergedSegments with a spread band.
io_tsinfer.py tsinfer front end: add_mutations, infer_tree_sequence.
io_singer.py SINGER front end: haploid VCF → posterior ARG samples; write_haploid_vcf, singer_tree_sequences.
io_rfmix.py RFMix comparator (genotype-native): VCF/map writers, run_rfmix, .fb.tsv/.msp.tsv parsers, rfmix_paint.
io_relate.py Relate --compress → tskit wrappers/checks (external C++ toolchain).
io.py The tspaint.io namespace — gathers the front ends.
compare.py Head-to-head harness: painters (tspaint_paint, nearest_reference_paint, rfmix_paint), score_painter, head_to_head.
validate.py Metrics (exposed as tspaint.metrics): accuracy, calibration, flicker, tract-boundary error, fragmentation, map_truth.
sim.py msprime admixture sims with census-event ground truth; simulate_admixture, local_ancestry_truth, admixture_demography.
ranked.py Order-only (dense-ranked) ablation (ranked_tree_sequence). Measured not beneficial (§6); kept runnable behind tspaint_paint(..., ranked=True).
diagnostics.py Front-end-agnostic premise checks: node-persistence histogram and edge-span summary — the go/no-go on the central premise (§5.1).
experiments.py End-to-end benchmark drivers (tspaint.experiments): sim → fit → paint → score, age/scaling sweeps, ensemble and SINGER drivers, the flicker metric.
