Setup

A few imports and small helpers: pops maps sample nodes to their (simulated) source population, write_vcz lays out a minimal VCF-Zarr the way bio2zarr’s vcf2zarr would, and two plotting helpers draw a painted posterior against the shaded truth.

def pops(ts):
    '''sample-node -> source name, and a {name: pop_id} lookup.'''
    npop = ts.tables.nodes.population
    name = {p: ts.population(p).metadata.get("name", str(p)) for p in range(ts.num_populations)}
    pid = {n: p for p, n in name.items()}
    samples_of = lambda nm: [int(s) for s in ts.samples() if npop[s] == pid[nm]]
    return name, pid, samples_of


def write_vcz(ts, path):
    '''Write a minimal VCF-Zarr (vcz) store from a mutated ts (stand-in for vcf2zarr output).'''
    import zarr
    G = ts.genotype_matrix()                       # (sites, samples), biallelic 0/1
    V, N = G.shape
    root = zarr.open(path, mode="w")
    def arr(nm, data, dims):
        root[nm] = data
        root[nm].attrs["_ARRAY_DIMENSIONS"] = dims
    arr("variant_position", np.asarray(ts.tables.sites.position).astype("i8"), ["variants"])
    arr("call_genotype", G[:, :, None].astype("i1"), ["variants", "samples", "ploidy"])
    arr("variant_allele", np.array([["0", "1"]] * V), ["variants", "alleles"])
    arr("variant_contig", np.zeros(V, "i4"), ["variants"])
    arr("contig_id", np.array(["1"]), ["contigs"])
    arr("sample_id", np.array([f"n{i}" for i in range(N)]), ["samples"])
    arr("variant_ancestral_allele", np.array(["0"] * V), ["variants"])
    return path


def track_xy(track, state):
    '''Step coordinates (x, P(state)) for a list of soft Segments.'''
    xs, ys = [], []
    for seg in track:
        xs += [seg.left, seg.right]
        ys += [float(seg.posterior[state])] * 2
    return np.asarray(xs), np.asarray(ys)


def plot_painting(ax, tracks, truth_segs, L, title, state=1):
    '''Shade the true `state` (B) tracts; overlay each painter's P(state=B).'''
    for (l, r, st) in truth_segs:
        if st == state:
            ax.axvspan(l / 1e3, r / 1e3, color="0.6", lw=0, alpha=0.5, zorder=0)
    for label, track in tracks.items():
        xs, ys = track_xy(track, state)
        ax.plot(xs / 1e3, ys, lw=1.6, label=label, alpha=0.6)
    ax.set(ylim=(-0.02, 1.02), xlim=(0, L / 1e3), title=title,
           xlabel="position (kb)", ylabel="P(ancestry = B)")
    ax.axhline(0.5, color="0.6", lw=0.6, ls=":")
    ax.legend(loc="center left")

Input

Real input is genotypes — a phased VCF or, at scale, a chunked VCF-Zarr. tspaint never asks you to build the tree sequence by hand: the two inference front ends take any of the three sources through one interface (source = a tree sequence, a .vcf[.gz], or a .zarr store) and hand back a tspaint-ready tree sequence.

Here we simulate an admixed cohort so we know the truth — 8 admixed individuals (the queries) plus pure A and B reference panels — overlay mutations, and write it out as a VCF (and a VCF-Zarr) exactly as if it had come off a sequencing pipeline.

⏺ Here’s each array in RateThroughTime, grounded in how the EM actually builds them (dating/em.py:253–262). The model is a time-inhomogeneous ancestry CTMC: the generator is piecewise-constant on a log-time grid of cells, cell k having Q(k) = [[-q_AB[k], q_AB[k]], [q_BA[k], -q_BA[k]]]. EM alternates an E-step that accumulates per-cell sufficient statistics (D, J, loglik) under the current rates, and an M-step that turns those into smooth rate curves (q_AB, q_BA). All the cell-indexed arrays share one time axis: centers.

┌────────────────┬──────────────────────────┬────────────────────────────────────────────────┐ │ array │ shape │ what it is │ ├────────────────┼──────────────────────────┼────────────────────────────────────────────────┤ │ centers │ (n_cells,) │ the time axis (generations ago) │ ├────────────────┼──────────────────────────┼────────────────────────────────────────────────┤ │ q_AB, q_BA │ (n_cells,) each │ the fitted directional rates — the deliverable │ ├────────────────┼──────────────────────────┼────────────────────────────────────────────────┤ │ D │ (n_cells, K) │ per-cell occupation (exposure) │ ├────────────────┼──────────────────────────┼────────────────────────────────────────────────┤ │ J │ (n_cells, K, K) │ per-cell directional jump counts │ ├────────────────┼──────────────────────────┼────────────────────────────────────────────────┤ │ loglik_history │ list, length = #EM iters │ log-likelihood per iteration │ └────────────────┴──────────────────────────┴────────────────────────────────────────────────┘

K = 2 (A=0, B=1), n_cells defaults to 40.

centers — the time grid (generations ago)

Cell centres on a log-spaced backward-time grid. log_time_grid builds n_cells+1 geometric edges spanning [min positive node age, 1.05 × max node age]; cell_centers is the geometric mean of adjacent edges (grid.py:14–26). Log spacing gives roughly equal branch-occupation per cell. Same units as the tree-sequence node times (generations). This is the x-axis you plot everything against.

q_AB, q_BA — the directional cross-ancestry rates (the answer)

One rate per time cell, evaluated at centers; units = transitions per generation. q_AB[k] is the A→B rate in cell k, q_BA[k] the B→A rate. Direction convention (from the code): the generator acts forward in time, parent→child = old→young, so a backward-time A→B admixture event appears in q_BA (em.py:35–37). Reading them: - where a curve rises with t = a divergence / gene-flow epoch at that age; - q_AB vs q_BA asymmetry = the direction of gene flow; - split_time reads the onset of q_AB + q_BA (em.py:69).

They’re the M-step output: essentially q_AB[k] ≈ smooth(J[k,0,1] / D[k,0]) and q_BA[k] ≈ smooth(J[k,1,0] / D[k,1]) — a Poisson rate (jumps ÷ exposure) fitted with a GCV-selected penalised spline over centers, floored at 1e-9, and flat-clamped outside the informative occupation window (mstep.py directional_rate_splines). So they’re smooth versions of the raw J/D ratio.

D — per-cell occupation / exposure (the denominator)

D[k, s] = total expected time lineages spend in ancestry state s inside time-cell k, accumulated over the whole ARG, span-weighted and edge-blocked (each tree-sequence edge banked once, weighted by its genomic span; §3.3). Built in the E-step from the Van Loan endpoint-conditioned dwell times, split across the cells a branch crosses (estep.py: D[k] += span · dwell). Its role is the rate denominator / exposure: D[:,0] is the A-time (denominator for q_AB, since A→B jumps only happen while in A), D[:,1] the B-time. Cells with negligible D are “no-data” — they define the informative window (M-step) and are the exposure guard in split_time. Units: generations × base-pairs (time × span).

J — per-cell directional expected jump counts (the numerator)

J[k, m, n] = expected number of m→n ancestry transitions occurring in time-cell k, span-weighted and edge-blocked, over the whole ARG. J[:,0,1] = A→B jumps (numerator for q_AB), J[:,1,0] = B→A jumps (numerator for q_BA); the diagonal is ~0. Also from the Van Loan endpoint-conditioned statistics (branch_stats), split across cells (estep.py: J[k] += span · jumps). A cell’s raw rate is J[k,0,1]/D[k,0] before the M-step smooths it.

▎ D and J are the sufficient statistics from the final E-step iteration — the raw, unsmoothed evidence; ▎ q_AB/q_BA are the smoothed rates fitted from them. If you want the noisy per-cell empirical rate (no ▎ spline), compute J[:,0,1]/D[:,0] yourself.

loglik_history — EM convergence trace

A list (one entry per EM iteration), not a cell array. Each entry is the span-integrated observed-data log-likelihood of the tip labels under the current Q(t) — the Felsenstein pruning likelihood summed over trees, span-weighted (estep.py: loglik += span · ll). It is non-decreasing (EM monotonicity); the loop stops when the relative change drops below tol (em.py:260). Use it to confirm the fit converged (flattening tail) and didn’t stall; len(loglik_history) tells you how many iterations ran (≤ n_iter).

How they fit together per iteration: E-step → (D, J, loglik) under the current rates → M-step smooths J/D into new q_AB, q_BA → repeat. centers indexes q_AB/q_BA/D(per state)/J(per state-pair); loglik_history indexes iterations.

One caveat that colours all of it: the time axis and rates are only as calibrated as the ARG’s node times (CLAUDE.md §6) — exact on an msprime true ARG, panmictic-prior-biased on Relate, coalescent-calibrated with SINGER. On an ensemble painting you get EnsembleRateThroughTime, where q_AB/q_BA are the member means and the split time carries a CI.

That coalescence time is a genealogical lower bound on a tract’s age: a segment where the query carries ancestry A cannot be younger than the point where its lineage last shared an ancestor with an A-source reference. So, per segment: take the query’s coalescence with the nearest reference of that segment’s ancestry, over the trees spanning the segment.

Concretely, from a Painting that still holds its ts + labels:

import numpy as np, tskit p = tspaint.paint(ts, labels) # segments() defaults to deadband=0.4 node_time = ts.tables.nodes.time refs = {s: [n for n, st in labels.items() if st == s] for s in set(labels.values())}

def segment_lower_dates(q): dated = [] for left, right, state in p.segments()[q]: # hard tracts for query q t = np.inf for tree in ts.trees(): lo, hi = tree.interval if hi <= left or lo >= right: # only trees overlapping the tract continue for ref in refs[state]: # nearest reference OF THIS tract’s ancestry if ref == q: continue m = tree.mrca(q, ref) if m != tskit.NULL: t = min(t, node_time[m]) dated.append((left, right, state, float(t) if np.isfinite(t) else np.nan)) return dated # (left, right, state, ≥ generations)

The shallowest same-ancestry coalescence across the tract is the tightest guaranteed lower bound; use the median over the tract’s trees instead of min if you want a point estimate rather than a floor.

Two caveats that matter here: - Calibration (CLAUDE.md §6). Raw node_time is only as trustworthy as the ARG’s branch lengths — exact on an msprime true ARG, panmictic-prior-biased on Relate. For real dates use a SINGER ensemble (coalescent-calibrated times; then average the per-member bounds), or work in the calibration-invariant rank depth (introgression.foreignness_track(…, depth=“time”|“rank”), introgression.py:11) if you only need relative ordering. - This is the nearest-reference version (foreignness_track gives the nearest reference of any label per locus); the snippet above restricts to the segment’s own ancestry, which is what you want for “how old is this A-tract.”

If this is a workflow you’ll use, I’d suggest wrapping it as a first-class method — e.g. Painting.segment_dates(deadband=…, reduce=“min”|“median”) returning (left, right, state, date) per query, with the SINGER-ensemble averaging built in. Want me to add that (source + tests, no notebooks)?

A single ARG cannot tell you how much to trust itself. The band widens where the posterior ARGs disagree (shown here for the query with the most disagreement) and collapses where they concur — calibrated honesty a single ARG (or an overconfident segment method) does not provide. Averaging buys calibration rather than raw accuracy: the merged painting’s mean confidence sits below a single sample’s, and on some seeds averaging-then-argmax even dilutes a confidently-correct call. The number of SINGER samples is therefore a knob you spend on uncertainty quantification, not on accuracy.

Honest caveat. Both methods are bounded by the genealogy, not by their own cleverness. On an unlucky region where the admixed haplotypes coalesce among themselves before the pulse, the query↔︎reference link is severed and both paintings fall to ~0.5 — the ancestry simply is not recoverable there (CLAUDE.md §9). The painting’s job is to report that as P ≈ 0.5, not to guess.

Audit the reference panel (reference QC)

Painting trusts the labels on the reference panel. Real panels are imperfect — some “reference” individuals are themselves admixed or mislabelled. tspaint.reference_qc audits the panel from the genealogy: it reads each reference’s leave-one-out osterior (what the rest of the tree says about it, ignoring its own label), ranks references by credibility, and maps each one’s own foreign tracts.

Recap

One toolkit, one progression:

Step Call What you get
Genotypes → ARG io.tsinfer / io.singer (VCF / Zarr / ts) a tspaint-ready tree sequence (or a posterior ensemble)
Paint & compare paint(ts, labels, smooth=True) calibrated soft P(ancestry) per haplotype per position
Quantify ARG uncertainty ensemble.merge_posterior_tables ensemble-mean painting + uncertainty band
Trust the panel reference_qc per-reference credibility + introgression maps
Hidden source (panel-relative) detect_ghost foreign-to-panel tracts + genome-wide burden
Hidden source (reference-free) detect_archaic deep archaic tracts with no archaic reference

The throughline is calibration: every output is a soft, honest posterior, so where the genealogy cannot decide you get P ≈ 0.5 and a wide band rather than a confident mistake — and the same leave-one-out machinery that paints queries audits references and surfaces sources that are not in the panel at all.

Full signatures are in the API reference.
