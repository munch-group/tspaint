SINGER — count = n_samples − burn_in

SINGER draws n_samples ARGs, thin iterations apart, then tspaint discards the first burn_in of them.

┌───────────────┬─────────────────────────────────────────────────────────────────┬───────────────┐ │ knob │ role │ affects │ │ │ │ count? │ ├───────────────┼─────────────────────────────────────────────────────────────────┼───────────────┤ │ n_samples │ how many draws SINGER writes │ yes │ │ (-n) │ │ │ ├───────────────┼─────────────────────────────────────────────────────────────────┼───────────────┤ │ thin (-thin) │ MCMC iterations between draws — SINGER’s own spacing │ no │ │ │ (decorrelation) │ │ ├───────────────┼─────────────────────────────────────────────────────────────────┼───────────────┤ │ burn_in │ leading draws tspaint drops │ yes │ └───────────────┴─────────────────────────────────────────────────────────────────┴───────────────┘

tree sequences = n_samples − burn_in. Confirmed: 6,burn2→4 · 8,burn3→5 · 5,burn4→1 · 4,burn4→0.

Returns a single TreeSequence when that’s 1, a list when >1, an empty list when ≤0. Total MCMC work ≈ n_samples × thin iterations.

Your showcase: ts_singer uses the defaults (n_samples=21, burn_in=20) → 1 ARG (a single ts); ts_singer_posterior (n_samples=24, burn_in=2) → 22 ARGs (an ensemble).

ARGweaver — count = ceil((S − burn_in) / thin), where S = floor(iters / sample_step) + 1

ARGweaver runs iters MCMC iterations and saves an ARG every sample_step iterations (→ S saved); tspaint then drops the first burn_in and keeps every thin-th of the rest.

┌──────────────────────────┬───────────────────────────────────────────────────────┬──────────────┐ │ knob │ role │ affects │ │ │ │ count? │ ├──────────────────────────┼───────────────────────────────────────────────────────┼──────────────┤ │ iters (-n) │ total MCMC iterations (not a sample count) │ yes │ ├──────────────────────────┼───────────────────────────────────────────────────────┼──────────────┤ │ sample_step │ ARGweaver’s spacing — the analogue of SINGER’s thin │ yes │ │ (–sample-step) │ (default 10) │ │ ├──────────────────────────┼───────────────────────────────────────────────────────┼──────────────┤ │ burn_in │ leading saved ARGs tspaint drops │ yes │ ├──────────────────────────┼───────────────────────────────────────────────────────┼──────────────┤ │ thin │ keep every thin-th, applied post-hoc by tspaint │ yes │ └──────────────────────────┴───────────────────────────────────────────────────────┴──────────────┘

tree sequences = ceil((S − burn_in) / thin). Confirmed: iters20,step5→5; +burn1,thin2→2;

iters20,step=None→3 (default step 10). Defaults (iters=100, sample_step=10, burn_in=0, thin=1) → 11 ARGs.

The trap (why it wasn’t clear)

  • thin means different things. In SINGER it’s the sampler’s inter-draw spacing (doesn’t change the count). In ARGweaver it’s a post-hoc keep-every-Nth (does change the count) — ARGweaver’s spacing knob is sample_step.
  • ARGweaver has no n_samples. Its count comes from iters / sample_step; SINGER’s comes directly from n_samples.
  • The docs were wrong. The SINGER docstring claimed n_samples default 20 (it’s 21) and burn_in default 5 (it’s 20) — so the default count looked like 15 when it’s actually 1.

What I changed (docs only, no behavior change)

Fixed the two wrong SINGER defaults and added an explicit count formula + worked examples to the Returns of both io.singer and io.argweaver, and clarified in each param doc which knob is the sampler’s spacing vs. a post-hoc subsample. Option tests still pass (21).

If the thin/sample_step asymmetry still bugs you, I could unify the surface — e.g. give ARGweaver an n_samples convenience (setting iters = n_samples × sample_step) so both read as “n_samples − burn_in”. That’s a small additive change; want it?
